# Vibe Coding 试做任务

> 这是一个一天的小项目，用来让我更全面地了解你的实际能力。请放心做，做得不完美没关系，**我更关心的是你如何思考、如何取舍、如何复盘**。

---

## 项目目标

做一个 **极简 Markdown 写作工具**，部署到公网（Vercel / Netlify / Cloudflare Pages 都行），我可以打开链接直接使用。

## 必做功能

1. **左右分栏**：左边 Markdown 编辑器，右边实时预览
2. **字数统计**：显示中英文字符数、预计阅读时间（中文按 300 字/分钟）
3. **自动保存**：内容保存到 localStorage，刷新页面不丢
4. **导出 HTML**：一键下载渲染后的 HTML 文件
5. **复制 Markdown**：一键复制原文到剪贴板
6. **移动端可用**：手机打开不能崩

## 加分项（有时间再做）

- 多份草稿管理（列表 + 切换）
- 深色模式
- 代码块语法高亮
- 任何你觉得有意思的小细节

## 技术栈

不限。React、Vue、Svelte、原生 HTML 都可以，挑你最熟的。

## 时间

**1 天**，从今天起算。建议实际投入 **4–6 小时**，不要熬夜爆肝，我想看你正常状态下的产出。

---

## 交付物

### 1. 在线 Demo

部署好的公网链接，我可以直接打开使用。

### 2. GitHub 仓库

- 代码 push 上去，README 简单说明怎么本地跑起来
- Commit 历史保留，不要 squash 成一个

### 3. 一份复盘文档（重要）

这部分我会认真看，**比代码本身更能反映你的工程素养**。请用 Markdown 写，不超过 1 页，至少回答以下问题：

1. **技术选型**：你为什么选这个技术栈？有没有考虑过别的？
2. **AI 使用**：这次 vibe coding 中，AI 在哪些地方帮了你，在哪些地方反而拖了后腿？你怎么发现并解决的？
3. **取舍**：哪些功能你做了，哪些你主动砍掉了？为什么？
4. **已知问题**：当前版本有哪些你已经发现但还没修的 bug 或体验问题？
5. **再给一天**：如果再给你一天时间，你会优先改进什么？为什么是这些？

---

## 一些重要的话

- **可以问问题**：开始之前如果觉得需求不清楚，**直接来问我**，不要猜。能问出好问题本身就是加分项。
- **可以用 AI**：这就是 vibe coding 的考察，Cursor / Claude Code / Copilot 随便用。
- **但要看得懂自己的代码**：交付后我会随机抽几段问你为什么这么写，所以不要采纳你不理解的代码。
- **诚实最重要**：如果某个功能没做完，老实写在复盘里。掩盖问题比有问题更糟糕。

---

## 评估方式

收到交付后，我会从这几个角度看：

- 产品体验和细节质量（空状态、错误处理、移动端）
- 代码可读性和你对自己代码的理解程度
- 复盘文档的深度和诚实度
- 整体完成度（不是越多越好，而是做了的东西是不是真的可用）
